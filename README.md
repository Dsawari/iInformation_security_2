# iInformation_security_2
Study assignment on the topic “Authentication and Access Management” for the subject “Fundamentals of Information Security”

1.Сотрудник получил SMS с кодом для входа в корпоративную почту. Сразу после этого ему позвонил человек, представившийся «техподдержкой», и попросил продиктовать этот код «для проверки сессии». Сотрудник согласился. Позже выяснилось, что аккаунт взломан.
Вопросы:
﻿﻿﻿Какой тип атаки был применён?
   - В данной ситуации был применен вид атаки "Фишинг".
﻿﻿﻿Почему SMS как второй фактор считается менее надёжным, чем приложение-аутентификатор?
   - Приложения для аутентификации безопаснее SMS-аутентификации, потому что оно генерируют коды двухфакторной аутентификации локально. Это не позволяет перехватывать коды, что возможно при использовании SMS. 
﻿﻿﻿Предложите два способа предотвратить такую утечку: один технический, один организационный.
   -   Технический способ: двухфакторная аутентификация, мониторинг подозрительных действий. Организационный способ: обучение сотрудников распознавать фишинг, регулярный аудит.

2.Задание:
Для каждой ситуации укажите, какая модель управления доступом подходит лучше всего: DAC, MAC, RBAC или АВАС.
Кратко объясните выбор.
Ситуации:
﻿﻿﻿Домашний компьютер: пользователь сам решает, какие папки расшарить в локальной сети.
   - Для данной ситуации больше всего подходит DAC, тк субъект (владелец) имеет право предоставлять другим пользователям доступ к аутентифицированным объектам или информации. Владелец контролирует права доступа к объекту. 
﻿﻿﻿Военная система с документами грифа
«Совершенно секретно».
   - Здесь подходит модель MAC, поскольку в рамках мандатной модели нельзя превышать назначенный уровень доступа. Если это необходимо, администраторы или владельцы систем создают для пользователей новые профили под актуальную метку ресурса.
﻿﻿﻿Крупный банк: 5000 сотрудников, чёткая иерархия должностей.
   - 
﻿﻿﻿Облачный сервис: доступ к документу зависит от должности, времени суток, геолокации и типа устройства.
﻿﻿﻿Небольшой стартап (10 человек), где все доверяют друг другу и задачи часто меняются.
